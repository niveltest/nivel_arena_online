# 修正計画：Renderサーバーでのソケット接続タイムアウト問題

## 現状の課題

- **接続タイムアウト**: Renderの無料プランでは、サーバーの起動（Cold Start）に時間がかかるため、デフォルトの15秒設定では接続が完了する前にタイムアウトしてしまうことがある。
- **デバッグ情報の不足**: サーバー側で接続試行が届いているか確認するためのログが不足している。

## 変更内容

### クライアント側 (`Lobby.tsx`, `GameBoard.tsx`)

- `socket.io-client` の `timeout` 設定を **15秒から45秒** に延長し、サーバーの起動時間を十分に確保する。

### サーバー側 (`index.ts`)

- サーバーバージョンを **v1.3** に更新。
- ソケット接続時に、現在の接続数を含むログを出力するように変更。

## 検証計画

- Render Dashboard でデプロイを監視し、`v1.3` が立ち上がるのを確認する。
- 数分放置してサーバーをスリープ状態にした後、アプリを開いて「CREATE ROOM」を試し、タイムアウトせずに接続できるか確認する。
