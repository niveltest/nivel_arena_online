# 公式Q&A準拠：詳細ルール修正計画

公式Q&Aの裁定に基づき、細かい挙動やエッジケースのロジックを修正しました。

## 実施内容

### 1. 「デュエリスト」(DUELIST) の制限強化

- **裁定**: 対向ユニットが存在する場合、隣接レーンの 〈ガーディアン〉(防壁) は発動できない。
- **実装済**: アタッカーが `DUELIST` を持ち、かつ正面にユニットがいる場合に、隣接防壁の割り込みを無効化するロジックを追加しました。

### 2. 「エントリー」(ENTRY) の発動条件拡大

- **裁定**: 手札以外（トラッシュやデッキ）からフィールドに出た場合も発動する。
- **実装済**: 現在の `ON_PLAY` を `ON_ENTRY` と整理し、カードがフィールドに配置されるすべてのタイミング（配置、蘇生等）でトリガーするように名称統一・拡張しました。

### 3. 「エグジット」(EXIT) のトリガー保証

- **裁定**: 効果だけでなく、コストとしての破壊でも発動する。
- **実装済**: `destroyUnit` を経由するすべての破壊において、`ON_EXIT` が確実に呼び出されるように整理しました。

### 4. 0パワーによるルール破壊 (Rule-based Trashing)

- **裁定**: パワーが0以下による破壊はルールの強制であり、「効果で破壊されない」耐性を無視する。
- **実装済**: パワー0での破壊時に、無敵などの耐性をバイパスしてトラッシュに送られるように `destroyUnit` の優先度を調整しました。

### 5. アイテムの相手ユニットへの装備

- **裁定**: 特記ない限り、相手のユニットにもアイテムを装備できる。
- **実装済**: `playCard` で敵陣のスロットも装備対象に選べるように修正しました。

### 6. 複数キーワードの最大値適用

- **裁定**: 同名の数値付きキーワードを複数持つ場合、最大値のみを採用する。
- **実装済**: `getKeywordValue` を修正し、加算ではなく `max` を取得するようにしました。
