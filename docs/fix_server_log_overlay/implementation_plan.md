# サーバーログUIの改善実装計画

サーバーログのオーバーレイが画面上の操作を妨げている（またはユーザーにとって邪魔になっている）問題を解決するため、以下のUI改善を行います。

## ユーザーレビューが必要な項目

- サーバーログをデフォルトで非表示にするか、あるいは最小化した状態で開始するか。
- デバッグ機能（FORCE WINなど）を隠しメニューにするか。

## 提案される変更

### [Component] GameBoard (components/GameBoard.tsx)

#### [MODIFY] [GameBoard.tsx](file:///c:/Users/worke/Antigravity/nivel_arena_online/components/GameBoard.tsx)

1. **表示状態の管理**:
    - `showDebugLogs` ステートを追加し、ログの表示/非表示を切り替えられるようにします。
2. **UIの改善**:
    - サーバーログのヘッダーに「最小化/展開」ボタンを追加します。
    - 最小化時は「LOGS」という小さなラベルのみを表示し、クリックで展開するようにします。
    - ログ表示エリアにスクロールバーを追加し、最大高度を制限（`max-h-[35vh]`など）することで、他のUI要素との干渉を減らします。
3. **フェーズ表示（ステータスバー）の移動**:
    - 現在画面最下部に配置されているステータスバーを、画面上部（プレイマットの上）に移動します。
    - これにより、手札の枚数が増えてもステータス情報が隠れないようになります。
4. **インタラクションの改善**:
    - ログコンテナを `motion.div` でラップし、ドラッグ可能にします（`drag` プロパティ）。

## 検証計画

### 手動確認

- サーバーログの「最小化」ボタンが正しく動作することを確認。
- ログを展開した時に、手札や他のプレイエリアを過剰に覆い隠さないことを確認。
- ドラッグ移動がスムーズに行えるか確認（ドラッグを導入する場合）。
