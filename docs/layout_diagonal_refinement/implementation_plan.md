# 実装計画 - 対角線配置の最適化とカードの向きの調整

ユーザーの要望に基づき、相手側のレイアウトを完璧な「対角線（点対称）」配置にし、かつプレイアビリティを向上させるためにカードの見え方を調整します。

## 概要

現在、相手側のプレイマット全体を `rotate(180deg)` することで対角線配置（点対称）を実現していますが、`DraggableZone` のインラインスタイルが優先されることでカードの向きが予期しない状態になっている可能性があります。これを修正し、配置の整合性を高めます。

## 変更内容

### [GameBoard.tsx](file:///c:/Users/worke/Antigravity/nivel_arena_online/components/GameBoard.tsx)

#### 1. `DraggableZone` の `transform` 修正

- 相手側（`isOpponent`）の場合、個別のゾーン内でさらに `rotate(180deg)` を適用するかどうかを検討します。
- 現在はコンテナ全体を回転させているため、カードは逆さまに見えます。
- **「対角線配置」を維持しつつ、カードを読みやすくするために、相手側のカードをさらに180度回転させて自分の方に向ける修正を行います。** これにより、位置は対角線上（右上が相手のリーダーなど）にありながら、カード自体は正対している状態になります。

#### 2. CSS 変数による制御の統合

- `transform` プロパティを `DraggableZone` 内で一貫して管理し、CSS クラスでの上書き問題を解消します。

## 検証計画

### 手動確認

- 自分のリーダーが左下にある場合、相手のリーダーが右上に配置されているか確認。
- 相手側のカードが（逆さまではなく）自分と同じ向きで表示され、読みやすくなっているか確認。
- 編集モードで相手側のレイアウトも連動して適切に移動するか確認。
