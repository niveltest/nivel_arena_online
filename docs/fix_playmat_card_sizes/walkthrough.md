# プレイマットとカードサイズ修正のウォークスルー

## 修正内容

### 1. レスポンシブ化 (Container Queries)

- `GameBoard.tsx`: プレイマットのコンテナ (`playmat-canvas`) に `container-type: size` を適用しました。
- 各カードスロット (`renderSlot`) およびデッキ、トラッシュ、スキル、ダメージゾーンのサイズ指定を、固定ピクセル (`w-28 h-40` 等) から `cqw` (コンテナ幅のパーセンテージ) ベースに変更しました。
  - メインスロット: `w-[13cqw] aspect-[5/7]`
  - その他ゾーン: `w-[10cqw] aspect-[5/7]`
- これにより、ウィンドウサイズを変更しても、カードサイズがプレイマットに対して一定の比率を保つようになりました。

### 2. レイアウト修正 (隙間の削除)

- `PlaymatArea` コンポーネントのラッパーから余分なパディング (`p-2 sm:p-4`) を削除しました。
- 相手と自分のプレイマットが密着し、一体感のあるレイアウトになりました。
- `flex-col` と `flex-1` の組み合わせにより、利用可能な垂直領域を最大限に使用しつつ、アスペクト比を維持しています。

### 3. カードコンポーネントの調整

- `Card.tsx`: 親コンテナからサイズ指定 (`className`) が渡された場合、デフォルトの固定サイズ (`w-16` 等) が適用されないようにロジックを微調整しました。

## 次のステップ

- ブラウザで実際の表示を確認し、必要であれば `cqw` の値を微調整して最適なバランスを見つけます。
