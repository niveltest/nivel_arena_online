# モバイル表示最適化計画

スマートフォン等の小型デバイスにおける視認性と操作性を向上させるための最適化を行います。
**※PCでの表示は現状を維持し、モバイル環境（画面幅が狭い場合）のみに調整を適用します。**

## 課題

- **全体**: デスクトップ向けの固定的なサイズ感により、画面端が見切れたり要素が重なったりしている。
- **マリガン画面**: カードが大きく表示され、スクロールしにくい。
- **ゲームボード**:
  - 手札のカードが大きく、フィールドを隠してしまっている。
  - フェイズ表示バーが密集しすぎている。
  - 「Next Phase」ボタンが相対的に大きすぎる。
- **ステータス表示**: モバイルではコストやパワーのアイコンサイズが相対的に大きく、隣のカードと被りやすい。

## 修正内容

### 1. 全体的なレスポンシブ対応 (`globals.css`)

- メディアクエリ (`@media (max-width: 640px)`) を導入。
- モバイル向けの変数調整を行い、全体のスケールを適正化します。

### 2. カードコンポーネントの調整 (`Card.tsx`)

- モバイル環境下でパディングやフォントサイズを縮小し、高密度な表示に対応します。
- ステータスアイコン（コスト、パワー等）を若干スケールダウンさせ、密集時の重なりを軽減します。

### 3. ゲームボードのレイアウト最適化 (`GameBoard.tsx`)

- **手札 (Hand)**: モバイルではスケールを下げるか、カード間の重なり（overlap）を強めて表示領域を確保します。
- **フェイズバー**: よりコンパクトなデザインに変更。
- **ヘッダー/ボタン**: フォントサイズやボタンサイズをモバイル向けに縮小。
- **マリガン表示**: グリッドの列数や余白を調整し、一度に多くのカードが見えるようにします。

## 検証プラン

- ビルドが通ることを確認。
- ブラウザのデベロッパーツールで iPhone 14 等の解像度（375x812）を選択し、以下の点を確認。
  - 全ての要素が画面内に収まっており、重なりが最小限であること。
  - カードの効果や数値が小さすぎず読めること。
  - ボタンが押しやすい位置・サイズであること。
